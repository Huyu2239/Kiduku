# Issue 作成フロー

## 目的

Issue の粒度と品質をそろえ、着手前に必要情報が揃った状態を作る。

## 人間による Issue 作成フロー

1. GitHub の `New issue` からテンプレートを選択する
2. 該当テンプレートの必須項目をすべて埋める
3. 既存 Issue と重複がないことを確認する
4. 機密情報が含まれていないことを確認して作成する

### テンプレートの使い分け

- `不具合報告`: 再現手順と期待挙動がある不具合
- `機能提案`: 課題と提案内容、受け入れ条件を定義する改善案
- `運用・保守タスク`: ドキュメント、CI、リファクタ等の非機能タスク
- `AI対話報告から起票`: 人間と AI の対話結果をもとに起票する場合

## AI エージェント連携フロー

人間がチャットで報告し、AI が内容を構造化して Issue を起票する。

1. AI が以下をヒアリングする
- 背景（何が問題か）
- 再現手順または要求内容
- 期待する結果
- 受け入れ条件
- 影響範囲

2. AI が `04-ai-dialog-report.md` 形式で草案を作る

3. 人間が草案を確認し、認識差分を修正する

4. AI が最終内容で Issue を作成する

### AI が起票時に満たす最低条件

- タイトルが一意で要点を表している
- 再現手順または要求仕様が明記されている
- 完了判定に使える受け入れ条件がある
- 機密情報が除去されている

## gh CLI での起票例

```bash
gh issue create \
  --title "[AI Report] 既読状況確認で空一覧時にエラー" \
  --body-file /tmp/issue-body.md
```

`/tmp/issue-body.md` は `04-ai-dialog-report.md` の見出し構成に合わせて作成する。
