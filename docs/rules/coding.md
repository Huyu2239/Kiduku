# コーディング規約（Rust）

## 目的

Rust コードの可読性と一貫性を保つ。

## 形式

- `cargo fmt` で整形する
- `cargo clippy` の警告は原則解消する

## 構造

- モジュール階層は浅く保つ
- 1ファイルにつき1責務を意識する
- データ構造は最小限に保つ

## エラーハンドリング

- 例外的な状況以外は `panic!` を使わない
- `Result` と `Option` を明示的に扱う
- 失敗理由は文脈が分かる形で返す

## 非同期

- ブロッキング処理は避ける
- 共有状態は最小限にする

## コメント

- 自明な処理にはコメントを付けない
- 意図や背景が伝わりにくい箇所のみ補足する
