# Docs Architecture

本ドキュメントは、`docs/` 配下に配置されるドキュメント群の構成方針と運用規約を定める。

## 基本方針

* `docs/` は **人間向けの正式ドキュメント** を格納する場所とする
* 設計・仕様・手順など、合意された内容のみを記述する
* AI専用、試行錯誤段階、内部メモは `.ai/docs/` に分離する

## ディレクトリ構成の原則

* `docs/` 直下には、用途別にディレクトリを配置する
* 各ディレクトリは責務が重ならないように設計する
* 新しいカテゴリを追加する場合は、既存構成との重複がないかを確認する

現在の想定構成は以下のとおりである。

* `onboarding/` : 開発を開始するための手順・前提
* `features/` : 機能仕様、要件、設計（PRD / DD 等）
* `architecture/` : 全体構造、設計方針、モジュール構成
* `rules/` : 開発・実装・運用に関する規約

## index.md について

* `docs/` および各サブディレクトリには `index.md` を配置する
* `index.md` は **索引専用** とし、詳細な説明は書かない
* 内容は以下に限定する

  * 配下ディレクトリへのリンク
  * 配下ファイルへのリンク
  * 各項目の簡潔な1文概要

## 記述ルール

* 1ファイルにつき、1つの主題を扱う
* 判断理由や背景が重要な場合は明示的に記述する
* 一時的な決定や未確定事項は、正式化するまで `docs/` に含めない

## 変更とメンテナンス

* `docs/` の内容はレビュー可能であることを前提とする
* 構成を変更する場合は、本ドキュメントを先に更新する
* 不要になったドキュメントは、削除または履歴が追える形で整理する

## 他ドキュメントとの関係

* `.ai/AGENTS.md` : 変更されない前提・制約を定義する
* `.ai/docs/` : AI専用の補助ドキュメントを格納する
* `docs/` : 人間向けの正式ドキュメントを格納する
