### 1. 企画概要

- Discord 既読確認Bot
- もう，見逃さない，見逃させない

### 2. 背景・課題

- 現状
    - Discordにはメンション機能がある
    - メンションメッセージを開くだけで既読がされる
    - 「未読にする」機能はあるが，実行工程がある
    - LINEのような既読数の表示がない
- 問題点・課題
    - 既読処理のハードルが低く，未読処理のハードルが高いため，重要なメッセージを見落としてしまうことがある
    - 誰が読んでくれたのかを把握できない

### 3. 目的・ゴール

- この企画で達成したいこと
    - 既読状況を可視化し、重要なメンションの見落としを減らすこと
    - チーム内で「誰が読んだか」を把握できる状態にすること
    - Discord の標準機能を損なわず、日常利用の負荷なく既読確認ができること
- 成功の指標（KPI など）
    - ユーザが，discordの使用体験を削ぐことなく，既読機能を利用することができる
- **想定される効果**
    - メンションの見落とし減少
    - チーム内の連絡ミスによる手戻りの減少

### 4. ターゲット

- 想定ユーザー・顧客
    - Discordを，チーム活動の場として利用しているサーバの管理者，メンバー
        - 大学のサークルやプロジェクトチーム
        - 社会人の小規模開発チーム
        - イベント運営用サーバ など
- ユースケース・利用シーン
    - サーバでのメンション時

### 5. 企画内容

- 提供する機能・サービス概要
    - 既読の取得方法
        - メンションを含むメッセージに Bot が「既読」リアクションを自動付与
        - 「既読」リアクションを押したユーザを既読とみなす
    - 未読ユーザへのフォロー
        - 未読のユーザに対し、N時間ごとに DM で通知して反応を促す
    - 送信者向けの確認機能
        - メンション送信者がアプリを実行すると、一時的メッセージで未読ユーザ一覧を表示
- 特徴・他との違い（差別化ポイント）
    - Discord標準機能では実現できない
