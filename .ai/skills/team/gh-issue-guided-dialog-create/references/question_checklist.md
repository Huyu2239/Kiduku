# Question Checklist

Use this checklist to collect minimum issue quality.

## Common (all issue types)

- Problem or request summary (1-2 sentences)
- Expected outcome
- Acceptance criteria (checkbox-ready)
- Impact scope (who/what is affected)
- Constraints and assumptions

## Bug-specific additions

- Reproduction steps
- Actual result
- Environment (OS/runtime/branch)
- Severity (high/medium/low)

## Feature-specific additions

- User pain/problem
- Proposed solution
- Alternatives considered
- Impact level

## Task-specific additions

- Objective
- Scope (in / out)
- Done definition
- Validation method
